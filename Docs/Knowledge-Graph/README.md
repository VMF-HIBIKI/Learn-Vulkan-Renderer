# 知识图谱

本目录记录渲染器实现过程中逐步形成的软件工程图谱。每完成一个可执行子任务，都必须更新对应图谱，让代码、任务计划和架构理解保持同步。

## 三层结构

1. [架构图](01-Architecture/README.md)
   - 记录系统如何分层、模块边界如何划分、每层使用了哪些主要库。
2. [任务类图](02-Class-Diagrams/README.md)
   - 记录每个任务新增或修改的主要类型、trait 实现、字段所有权和依赖关系。
3. [任务时序图](03-Sequence-Diagrams/README.md)
   - 记录每个任务的运行时调用顺序、事件流、生命周期和资源交接。

## 更新规则

- 每个任务至少更新一张对应任务图。
- 如果任务引入新库、跨模块边界或新的系统分层，必须更新第一层架构图。
- 如果任务新增 struct、trait、resource、handle owner 或重要封装，必须更新第二层类图。
- 如果任务新增运行流程、事件调度、GPU 提交、资源创建/销毁顺序，必须更新第三层时序图。
- 图使用 Mermaid，优先使用软件工程中的 `classDiagram` 和 `sequenceDiagram`。
